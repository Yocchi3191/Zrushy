特定のタイミング、イベントによって突入するゲームモードで、ヒロインの部位に触れることで開発度や好感度を稼ぐことができる。

各パラメータが一定以上になることで、特殊なイベントや新しいおさわりが解放される。

## パラメータについて

### 快感

開発度と好感度に比例する

**【実装に必要な情報】**
- [ ] 快感の計算式（例: 快感 = 開発度 × 係数A + 好感度 × 係数B）
- [ ] 快感の最大値・最小値
- [ ] 快感が一定値に達したときの挙動（絶頂など）

### 開発度

おさわりや特定のイベントで増える

部位ごとに蓄積され、溜まる量はさわり方によって異なる

感じやすい他の部位や行為と合わせてさわることで「条件付け」が発生して、より高い開発度を稼ぐことができる

**【実装に必要な情報】**
- [ ] 開発度の最大値（部位ごと？全体共通？）
- [ ] 各部位 × 各さわり方の基本増加量（マスターデータ化が必要）
  - 例: 胸を揉む → 開発度+5、胸を舐める → 開発度+3
- [ ] 「条件付け」の具体的な仕組み
  - どの部位とどの部位の組み合わせで発生するか
  - 条件付けボーナスの計算方法（倍率？固定値？）
  - 条件付けの状態管理方法（永続的？一時的？）

### 好感度

- 増える
    - 生活パートでのやりとり
    - 覚醒状態でのおさわり
- 増えない
    - 非覚醒状態のおさわり(ex. すやすや、気絶中など)

**【実装に必要な情報】**
- [ ] 好感度の最大値
- [ ] 覚醒状態でのおさわり時の好感度増加量
  - さわり方による違いはあるか？
  - 部位による違いはあるか？
- [ ] 覚醒/非覚醒の状態判定方法（すやすや度が閾値以下？専用フラグ？）

### すやすや度

ヒロインの意識の覚醒度合い。すやすやおさわりなどの状態で主に使用する。

おさわりによって上昇し、最大までいくと覚醒する。

覚醒後は好感度、えっち興味度によって継続判定を行う。

**【実装に必要な情報】**
- [ ] すやすや度の最大値
- [ ] おさわりによるすやすや度の増加量
  - さわり方による違いはあるか？
  - 部位による違いはあるか？
- [ ] 覚醒の閾値（最大値？それとも最大値の何%？）
- [ ] 継続判定の具体的な条件式
  - 例: 好感度 >= X かつ 性欲 >= Y なら継続
  - 判定失敗時の挙動（拒否？怒り？）

### 性欲

えっちなことがどれだけしたいか

「スケベブック」などのえっち興味度を上げるアイテムやイベントによって増減する

おさわりによって絶頂した場合、性欲は必ず減少する。減少量は好感度、開発度によって変化する

**【実装に必要な情報】**
- [ ] 性欲の最大値・最小値
- [ ] 性欲の初期値
- [ ] 絶頂時の性欲減少量の計算式
  - 例: 減少量 = 基本値 - (好感度 × 係数A) - (開発度 × 係数B)
- [ ] アイテム/イベントでの増減量（マスターデータ化）

## パラメータの管理方法

**【実装に必要な情報】**
- [ ] パラメータはどこに保存されるか？
  - Body全体で管理？
  - Part（部位）ごとに管理？
  - 両方のハイブリッド？
- [ ] パラメータの永続化方法
  - セーブデータに含めるか
  - セッション内のみか

## リアクションとイベントの発生条件

**【実装に必要な情報】**
- [ ] リアクションの発生条件
  - パラメータの範囲で決まる？（例: 開発度10-20なら「恥ずかしそう」）
  - さわり方とパラメータの組み合わせ？
  - マスターデータでの管理方法
- [ ] イベントの発生条件
  - 特定のパラメータが閾値を超えたとき？
  - 特定の部位 × さわり方 × パラメータ範囲の組み合わせ？
  - イベント発生後の挙動（パラメータリセット？継続？）
