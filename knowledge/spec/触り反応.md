## 流れ

1. 部位にさわる
2. 部位に対してポイントがたまる
   - 快感、開発度、好感度 など
3. さわった部位の、現在のパラメータ状態で返すリアクションを取得
4. 〃で発生するイベントを取得
5. リアクションを再生する
6. イベントを再生する

## さわり方の種類

### 口

- キス
- 舐める
- しゃぶる
- 咥える
- 舌を絡める
- 噛む

### 手

- 撫でる
- 揉む
- つまむ
- まさぐる

### おちんちん

- 擦る
- 挿れる
  - 「こういう挿れ方が用意されている」ではなく、「ある深さ・位置で動かしたときに対応する部位へのさわり反応が起こる」設計にする

### 全身

- 抱きしめる

**実装方針:**

- さわり方の識別: Strategy パターンの使用を検討

## 部位の定義

詳細は[[部位一覧]]を参照。

**仕様:**

- 部位は階層構造を持つ（例: 胴 → おっぱい → 乳首）
- 親部位に当たり判定があるものとないものがある
  - 当たり判定なし: 胴
  - 当たり判定あり: おっぱい

**実装に必要な情報:**

- 部位ごとのパラメータ管理
  - どの部位が独自の開発度を持つか
  - 部位間の関連性（条件付けのため）

## おちんちんの「挿れる」実装について

> 「ある深さ・位置で動かしたときに対応する部位へのさわり反応が起こる」設計

### 深さ・位置の表現方法

- **段階表現を採用**
  - 腟口 → 腟内 → G スポット → 子宮口

### 位置（角度・方向）の表現

- **G スポット**: 角度付き入力でさわらせたい
- その他の部位: 深さのみで判定

### 動きの検出方法

- **「この深さに移動する」を取得する**
  - 「この深さで動く」は表現が難しいため、移動イベントベースで実装

**実装に必要な情報:**

- [ ] 深さ段階と部位のマッピング定義
- [ ] G スポットの角度判定仕様
- [ ] 移動イベントの検出・通知方法

## リアクションの種類と内容

### リアクションに含まれる要素

- テキスト（セリフ）
- 表情
- アニメーション
- 効果音
- その他

### リアクションのバリエーション

- **同じ条件で複数のリアクションからランダム選択**
- **履歴による変化**: 特定のリアクションの発火条件として使用したい

**実装に必要な情報:**

- [ ] リアクションの選択ロジック（重み付けランダムなど）
- [ ] 履歴の保持方法と参照方法
- [ ] 発火条件の定義方法

## イベントの種類と内容

### イベントの具体例

- 初めて特定の部位をさわった
- 開発度が閾値を超えた
- 絶頂に達した
- 条件付けが完成した
- その他

**実装に必要な情報:**

- [ ] イベントの発生タイミング
  - さわった瞬間？
  - パラメータ更新後？
  - リアクション再生後？
- [ ] イベントの内容
  - カットシーン？
  - 特殊なリアクション？
  - パラメータの大幅変化？
  - フラグの設定？
- [ ] イベントの発生制御
  - 一度きり？
  - 条件を満たすたび毎回？
  - クールダウン期間あり？

## 同時さわりの扱い

**【実装に必要な情報】**


- [ ] 複数の部位を同時にさわれるか？
  - 例: 片手で胸、片手で性器
  - 例: 口でキス、手で胸
- [ ] 同時さわりの場合のパラメータ計算
  - 単純加算？
  - 相乗効果（条件付け）？
  - どちらか一方のみ？
- [ ] 同時さわりのリアクション
  - 優先度の高い方のみ？
  - 専用の組み合わせリアクション？
