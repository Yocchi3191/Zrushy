## 流れ

1. 部位にさわる
2. 部位に対してポイントがたまる
    1. 快感、開発度、好感度 とか
3. さわった部位の、現在のパラメータ状態で返すリアクションを取得
4. 〃で発生するイベントを取得
5. リアクションを再生する
6. イベントを再生する

## さわり方の種類

- キス
- 舐める
- しゃぶる
- 咥える
- 舌を絡める
- 噛む

### 手

- 撫でる
- 揉む
- つまむ
- まさぐる

### おちんちん

- 擦る
- 挿れる
    - 浅く
    - 深く
    - 「こういう挿れ方が用意されている」ではなく、「ある深さ・位置で動かしたときに対応する部位へのさわり反応が起こる」設計にする

## 部位の定義

**【実装に必要な情報】**

- [ ] さわれる部位の一覧
    - 例: 頭、顔、口、首、胸（左右別？）、お腹、背中、腰、お尻、太もも、膝、足、性器（詳細部位）など
- [ ] 部位の階層構造
    - 例: 胸 → 乳首、胸全体
    - 例: 性器 → クリトリス、膣口、膣内（浅い/深い）
- [ ] 部位ごとのパラメータ管理
    - どの部位が独自の開発度を持つか
    - 部位間の関連性（条件付けのため）

## おちんちんの「挿れる」実装について

> 「ある深さ・位置で動かしたときに対応する部位へのさわり反応が起こる」設計

**【実装に必要な情報】**

- [ ] 深さ・位置の表現方法
    - 数値（0.0〜1.0 の正規化？cm 単位？）
    - 離散的な段階（浅い/中間/深い）？
- [ ] 深さ → 部位のマッピング
    - 例: 深さ 0-3cm → 膣口、深さ 3-7cm → 膣内浅い、深さ 7+cm → 膣内深い
- [ ] 位置（角度・方向）の表現は必要か？
    - 前壁、後壁など
- [ ] 動きの検出方法
    - 速度、振幅、リズムなども考慮するか
    - それともシンプルに「この深さで動いている」だけ？

## イベントの種類と内容

**【実装に必要な情報】**

- [ ] イベントの具体例
    - 初めて特定の部位をさわった
    - 開発度が閾値を超えた
    - 絶頂に達した
    - 条件付けが完成した
    - その他
- [ ] イベントの発生タイミング
    - さわった瞬間？
    - パラメータ更新後？
    - リアクション再生後？
- [ ] イベントの内容
    - カットシーン？
    - 特殊なリアクション？
    - パラメータの大幅変化？
    - フラグの設定？
- [ ] イベントの発生制御
    - 一度きり？
    - 条件を満たすたび毎回？
    - クールダウン期間あり？

## 同時さわりの扱い

**【実装に必要な情報】**

- [ ] 複数の部位を同時にさわれるか？

    - 例: 口でキス、手で胸
- [ ] 同時さわりの場合のパラメータ計算
    - 単純加算？
    - 相乗効果（条件付け）？
    - どちらか一方のみ？
- [ ] 同時さわりのリアクション
    - 優先度の高い方のみ？
    - 専用の組み合わせリアクション？

## マスターデータの設計

**【実装に必要な情報】**

- [ ] データの管理方法
    - JSON/CSV/ScriptableObject？
    - データベース？
- [ ] 必要なマスターデータテーブル
    - 部位マスター（部位 ID、名前、初期パラメータなど）
