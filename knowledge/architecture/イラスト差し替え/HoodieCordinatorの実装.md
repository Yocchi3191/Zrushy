## 仕様

そもそも、フード周りのイラスト差し替えの仕様について

- フードは、`服右`、`服左`、`ジッパー`の 3 部位で構成される
- 各部位はイラストの状態を持つ
- ジッパー: 上がり <-> 半分 <-> 下ろし
- 服右: 閉じ <-> 半開 <-> 開き
- 服左: 閉じ <-> 半開 <-> 開き
- 状態遷移は 3 パーツすべて、となりあった状態にのみ遷移する
- 服右/左の取りうる状態は、ジッパーの状態に制限される
- 具体的には
    - ジッパーが`上がり`のとき、服右/左は`閉じ`より先の状態には遷移できない
    - ジッパー`半分` => 服右/左 `半分` まで
    - ジッパー `下がり` => 服 `開き`まで
- ジッパーの状態が遷移したとき、服の状態が条件に違反した場合、現在の状態から最も近い、制約を満たす状態まで遷移する
    - 強制遷移が発生するのは制約が厳しくなる方向（下がり → 半分、半分 → 上がり、下がり → 上がり）のみ
    - 制約が緩む方向（上がり → 半分、半分 → 下がり）は違反が発生しないため何もしない
    - 例: ジッパー `下がり`: 服右/左: `開き`
    - ジッパー `半分`に遷移
    - 服が条件違反
    - 服右/左: `半開`に強制遷移
- 服右と服左は独立している（それぞれ別々に状態を持つ）
