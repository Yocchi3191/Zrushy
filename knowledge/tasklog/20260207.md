# yarn spinner経由でシナリオ進行を制御する

部位クリックでシナリオ進行できるようになった
でも思ってたんと違う

やりたい挙動

- 頭クリック => 頭さわり開始 => head_default#1
- しばらく頭さわりを継続 => head_default#2
- 頭さわりをさらに継続 => head_default#3

っていう流れにしたい
クリックして進行、じゃなくて、クリックしてエンティティにパラメータを反映していって、特定条件になったら次のAction, ラインが渡される

例えば
1. おさわり継続によるシナリオ更新
2. パラメータが一定値になったときにシナリオ更新
3. 特定の部位を触ったときにシナリオ更新
	1. 発火は部位ごとのシナリオで設定すればいいので、ここでのシナリオ更新は「頭→右腕」などのさわり部位の変更があたる

質問返答

 1. 更新トリガーの定義場所: yarnファイルで設定できるのが最も柔軟に設定できる？
 2. トリガー未定義シナリオ: 1アクションで終了させるようにする。逆に言うと更新の必要なシナリオは必ずトリガーが必要？
 3. おさわり継続とは: 「おさわり継続」イベントに部位別のさわり回数を記録するオブジェクトを注入して、触り回数を取得できるようにする
	 1. 他のゲームだと、一度クリックした部位に自動的におさわりをし続けるシステムがあるので、クリック回数での判定ではなく、「おさわり」の回数を数えたい
	 2. 進行停止条件: 考えられていない。一定時間が妥当そう
	 3. 間隔: 間隔とは？待ち受け時間や、ミスおさわり(シナリオ更新に無関係な部位へのおさわり)の回数は使いたい。どちらかというとミスおさわり回数のほうが安定して判定できそう
