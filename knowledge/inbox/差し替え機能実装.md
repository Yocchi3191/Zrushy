## 入力からイラストパーツ取得の導線

決定事項:

- UseCase の引数は表情タイプのみ（UI 側はパーツ構造を知らない）
- スプライトレイヤーの識別子は `SpriteLayerId`（`PartID` とは別概念）
- `SpriteLayerId` は Application 層で宣言（Presentation・UseCase の両方が参照できる）
- VM は既存の `HeroinViewModel` に `Dictionary<SpriteLayerId, string>` を追加する

1. ボタンクリック
2. Controller → `ChangeSpriteLayerUseCase.Execute(expressionType)`
    - 戻り値: `Dictionary<SpriteLayerId, string>`（レイヤーごとのアセットパス）
    - 検索失敗は exception で OK
3. `SpriteLayerRepository.Find(expressionType)` → `Dictionary<SpriteLayerId, string>` を返す
    - アセット実体ではなくパスを返す
4. Controller が戻り値を `HeroinViewModel` に渡して更新
    - `HeroinViewModel.OnUpdated` 発火
    - `HeroinView` が購読してスプライトを差し替え

SpriteLayerRepository を実装する

- 特定位置に画像ディレクトリがある
- Get()したらディレクトリからファイル探してくる
- あったらパス返す

```mermaid
classDiagram

namespace core{
	class SpriteLayerID
	class ExpressionType

	class SpriteLayerController{
		ChangeSprite(ID, Type)
	}
	class ChangeSprite{
		Execute(ID, Type)
	}
	class ISpriteLayerRepository{
		Get(ID, Type)
	}
	class SpriteLayerRepository{
		Get(ID, Type)
	}
	class HeroinViewModel{
		OnUpdated()
		SpritePaths: Dictionary<SpriteLayerId, string>
	}
}

namespace unity{
	class HeroinView{
		OnUpdated()
	}
	class Imageコンポーネント
	class SpriteLayerBinding{
		SpriteLayerId
		Imageコンポーネント
	}
}

SpriteLayerController --> ChangeSprite : 画像パスもらってくる
ChangeSprite --> ISpriteLayerRepository
ISpriteLayerRepository <|.. SpriteLayerRepository
SpriteLayerController --> HeroinViewModel : 更新
HeroinViewModel <-- HeroinView : 監視・更新

HeroinView *-- SpriteLayerBinding : ID で紐づけ
SpriteLayerBinding --> Imageコンポーネント : スプライト差し替え
```

処理の流れ

```mermaid
flowchart TD
    A[ボタンクリック] --> B[SpriteLayerController.ChangeSprite]
    B --> C[ChangeSprite.Execute]
    C --> D[SpriteLayerRepository.Get]
    D --> E[パス返却]
    E --> F[HeroinViewModel.UpdateSprite]
    F --> G[HeroinView.OnViewModelUpdated]
    G --> H[Resources.Load / Image更新]
```
