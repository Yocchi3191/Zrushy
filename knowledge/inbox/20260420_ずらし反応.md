# 服ずらし時の反応・イベント発火の実装

## 目的

服ずらしの際に、特定のイベントを発火させることで、ゲーム内の反応や演出を実装すること

## 設計決定

### 方針

- `InteractPart`（おさわり）と並行する構造にする
- 成功/失敗の2値で異なるシナリオを発火
- `IEventRepository` / `EventBus` は既存のものを流用

### ユースケース整理

- `CanZrushy` は廃止。`ZrushyClothing`（実行 + 結果返却）に一本化
  - 旧: `bool CanZrushy(ZrushyInput)` → 判定のみ
  - 新: `ZrushyResult Execute(ZrushyInput)` → 判定 + イベント発火 + 結果返却

### 成功/失敗の扱い

- `ZrushyResult` (enum: Success / Failure) で2値表現
- リポジトリ側で `GetEvents(ClothingID, ZrushyResult)` に応じてイベントリストを分ける
- 各イベントは既存の `ICondition[]` で追加条件も設定可能

### コールチェーン

```
Zrushable.OnEndDrag
  → IZrushyClothing.Execute(ZrushyInput) : ZrushyResult
       → Heroin.CanPutOffClothing(ClothingID) → bool
       → ZrushyResult = bool ? Success : Failure
       → IClothingEventEvaluator.Evaluate(ClothingID, ZrushyResult)
            → IZrushyHistory.Record(ClothingID, ZrushyResult)
            → IClothingEventRepository.GetEvents(ClothingID, ZrushyResult)
            → IScenarioEvent.CanFire() フィルタ → Priority ソート
            → EventBus.Publish(fired)
       → return ZrushyResult
  → result == Success → _spriteInputHandler.TryTransition(input)
```

### 新規コンポーネント

| コンポーネント | レイヤー | 配置 |
|---|---|---|
| `ZrushyResult` | Domain VO | `Domain/Interactions/ValueObject/` |
| `IClothingEventEvaluator` | Domain | `Domain/Events/Service/` |
| `IClothingEventRepository` | Domain | `Domain/Events/Repository/` |
| `IZrushyHistory` | Domain | `Domain/Interactions/Repository/` |
| `ClothingEventEvaluator` | Application | `Application/` |

### 変更・削除

- `IZrushyClothing.CanZrushy` → `Execute` にリネーム、戻り値を `ZrushyResult` に変更
- `ZrushyClothing` の実装を更新（`IClothingEventEvaluator` を注入）
- `Zrushable.cs`: `Execute` の結果が `Success` のときのみ `TryTransition` を呼ぶ
