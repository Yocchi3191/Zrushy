## issue #16: 入力パターンを増やす

---

## 現状把握

- `InteractionType` enum は**既存**（`Finger`, `Penis`）
  - `Interaction` record が `InteractionType` を持つ（デフォルト `Finger`）
  - `SecretPart.CalculateArousal()` がすでに `Finger` / `Penis` で分岐済み
- **問題**: `PartInput` は `PartID` のみ → UI からの入力種別がドメインに届いていない
- `Clickable` は `IPointerClickHandler` のみ → クリック以外未対応
- 入力に対するパラメータ変化ロジックは **部位側（IPart）** に持つ（確定）

---

## 実装方針

### InteractionType マッピング

| 入力             | InteractionType        | イメージ                                                    |
| ---------------- | ---------------------- | ----------------------------------------------------------- |
| 左クリック(短)   | `Finger`（既存・維持） | 通常おさわり（指/手での接触）                               |
| 左クリック(長)   | `Press`                | 押し込む、入れ込む、つねる                                  |
| ドラッグ         | `Stroke`               | 服をずらす、擦る                                            |
| 右クリック（短） | `Tongue`               | 舐める、キス                                                |
| 右クリック（長） | `Oral`                 | しゃぶる、ディープキス                                      |
| スクロール       | `Lick`                 | 素早く舐め上げる（※`Tongue`と意味が近いので再検討余地あり） |

> `Finger` は「通常おさわり」の意味で維持。`SecretPart` への影響を避けるためリネームしない。

### PartInput の変更

`PartInput(PartID)` → `PartInput(PartID, InteractionType)`
影響範囲: `PartController.SendInput()` / `InteractPartCommand` / `Clickable`

### Clickable の拡張

| ジェスチャー     | 対応インターフェース             | 備考                                        |
| ---------------- | -------------------------------- | ------------------------------------------- |
| 左クリック       | `IPointerClickHandler`（既存）   | `eventData.button == Left`                  |
| 右クリック（短） | `IPointerClickHandler`           | `eventData.button == Right`、長押し判定なし |
| 右クリック（長） | `IPointerDownHandler` + 時間計測 | 閾値以上で `Oral` に変更                    |
| ドラッグ         | `IDragHandler`                   | 発火タイミング要検討（後述）                |
| スクロール       | `IScrollHandler`                 | `OnScroll()`                                |

**ドラッグの発火タイミング（未決定）**

- 毎フレーム発火（`OnDrag`）
- 一定距離ごと発火
- `OnEndDrag` 時のみ

### Input System (.inputactions)

既存の `Attack`（左クリック）は使わず、新規アクションを追加する。

## スコープ外（別 issue）

### Penisパーツ入力

画面上に Penis パーツを配置してドラッグで挿入・ピストンする仕組みにしたい。
「マウスジェスチャー → InteractionType」とは別の入力経路（専用コンポーネント＋接触判定）が必要なため、今回のスコープには含めない。

### さわりパーツの配置 UI

胸クリック → 手パーツを配置 → 再クリックでアニメーション、という仕組みも同様に別 issue。
現在の実装は「触り方モード」を状態として保持しないため、大きな設計変更が必要。

### VirtualCursor の見た目変更

触り方によってカーソル（手スプライト）のアニメーションを変えたいが、
入力に対して即座に InteractionType が決まる現在の設計では「モード変化イベント」を持てない。
今後の設計次第。

## 今後追加したい部位（参考・別 issue）

| 部位     | 優先度 | 対応インタラクション                             |
| -------- | ------ | ------------------------------------------------ |
| 口       | high   | キス（`Tongue`）、ディープキス（`Oral`）、フェラ |
| 手       | high   | 握る、手淫                                       |
| 乳首     | mid    | 乳房とは別に独立した部位として                   |
| 耳       | mid    | 触る（`Finger`）、舐める（`Tongue`）             |
| 頭（髪） | mid    | 撫でる（`Stroke`）                               |
