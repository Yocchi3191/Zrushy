## めも

5回はイカせたいけど、時間制限になって2,3回しかイカせられないで1日が終わる。
そんなゲームにしたい。

そのためには豊富なイカせ反応が必要。いろんなインタラクションがイカせルートに繋がっていることも大切。
絶頂パターンは部位ごとに違うはずだし、同じ部位の絶頂パターンも違うパターンがあったり、イッた回数が増えて疲れてくると余裕が無くなったりするべき。

クリッカーだから基本はVでなんとかなる。ずらし、クリックとかのインプットハンドリングもVでやる
Mは何をするか。部位ごとの開発度であったり、性癖みたいなマスクデータを貯める。
VへのインプットはVへのフィードバックとMへのデータ送信の2つを同時に行う。Mの状態に応じてVのフィードバックも変わる。そんな設計になったらだいぶ面白いと思う。

まずはViewでマウスインプットごとのインタラクションを作ればいいと思う。そこからリアクションのバリエーションを取るために、Mをどんどん充実させていけばよさそう。
